1- SNMP scan
```powershell
# nmap  
sudo grc nmap -sU -p 161 --script "snmp* and not snmp-brute" <% tp.frontmatter["RHOSTS"] %>
```


2- SNMP community enum
```powershell
# quick enum  
onesixtyone -c /usr/share/seclists/Discovery/SNMP/common-snmp-community-strings-onesixtyone.txt <% tp.frontmatter["RHOSTS"] %> -w 100  
  
# extended enum  
hydra -P /usr/share/seclists/Discovery/SNMP/snmp.txt -v <% tp.frontmatter["RHOSTS"] %> snmp
```

3-  SNMP basic enum
```powershell
# comprehensive enumeration (system/network/process/software info)  
snmp-check <% tp.frontmatter["RHOSTS"] %>  
  
# basic enumeration with onesixtyone, using default 'public' community string  
onesixtyone <% tp.frontmatter["RHOSTS"] %> public
```

4- SNMP get all available info

```powershell
snmpwalk -v2c -c public <% tp.frontmatter["RHOSTS"] %>
```

```powershell
subl nsExtendObjects
```

```powershell
snmpset -m +NET-SNMP-EXTEND-MIB -v 2c -c <read-write-community-string> \  
host-with-snmpd.lan \  
'nsExtendStatus."command"' = createAndGo \  
'nsExtendCommand."command"' = /bin/echo \

```

run it against the service 
```powershell
snmpwalk -v2c -c public <% tp.frontmatter["RHOSTS"] %> host-with-snmpd.lan nsExtendObjects
snmpwalk -v2c -c public <% tp.frontmatter["RHOSTS"] %> nsExtendObjects
```
